Ты должен проверить и исправить ВСЕ ошибки в Schema.org разметке моего сайта, настроить favicon для отображения в поисковиках, и сделать так чтобы сайт находился по ЛЮБЫМ запросам связанным с грузоперевозками.

Сайт: https://almik.onrender.com
Файлы которые можно менять: client/index.html, client/src/App.tsx, client/public/robots.txt, client/public/sitemap.xml, client/public/manifest.json
Файл который НЕЛЬЗЯ трогать: client/src/pages/Home.tsx

ЧАСТЬ 1: ИСПРАВИТЬ ОШИБКИ SCHEMA.ORG
В файле client/src/App.tsx есть Schema.org разметка. Валидатор показал 3 ошибки. Исправь их:

Ошибка 1: CallAction не поддерживается
В LocalBusiness схеме замени ВСЕ CallAction на ContactAction. Было:

json
{
  "@type": "CallAction",
  "name": "Позвонить сейчас",
  "target": { "@type": "EntryPoint", "urlTemplate": "tel:+79011172371" }
}
Должно стать:

json
{
  "@type": "ContactAction",
  "name": "Позвонить сейчас",
  "target": { "@type": "EntryPoint", "urlTemplate": "tel:+79011172371" }
}
Ошибка 2: TransportService не существует
В файле client/index.html есть скрытая разметка с itemtype="https://schema.org/TransportService". Такого типа НЕТ в Schema.org. Удали этот блок полностью:

html
<div style="display: none;" itemscope itemtype="https://schema.org/TransportService">
    <span itemprop="name">Грузоперевозки Тверь АЛМИК</span>
    ...
</div>
Ошибка 3: geo в Organization
В Organization схеме НЕЛЬЗЯ использовать свойство geo. Оно работает только в LocalBusiness. Найди в organizationSchema и удали этот блок:

json
"geo": {
  "@type": "GeoCoordinates",
  "latitude": 56.8587,
  "longitude": 35.9176
},
После исправлений проверь что JSON валидный (все скобки закрыты, все запятые на месте, нет лишних запятых перед закрывающими скобками).
